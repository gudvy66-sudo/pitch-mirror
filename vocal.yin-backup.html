🔍 보컬 버릇 진단 (검출 데이터 기반)

    📱 오너 테스트 방법

    • 이 파일을 GitHub Pages(pitch-mirror)에 올리고 폰 브라우저로 열기 (마이크는 HTTPS 필수)
    • 따라부르기: 목표 멜로디 듣기 → 녹음 → 같은 음 따라 부르기 → 분석
    • 확인 포인트: 음정 곡선이 내가 부른 것과 비슷한가 / 구간별 오차가 말이 되나 / 버릇 진단이 실제 내 습관과 맞나
    • 같은 멜로디 2번 녹음해서 "전/후 점수"가 개선을 반영하나 확인

    ⚠️ 이 방식의 정직한 한계

    • 반주 있으면 안 됨(단선율만). MR/피아노 깔고 부르면 분석 망가짐 — 목소리만.
    • 저음 옥타브 튐. 남성 저음(~80Hz)은 한 옥타브 위/아래로 잘못 잡힐 수 있음 → 곡선에 튄 점으로 보임.
    • 마이크·거리·소음 의존. 폰마다 결과 다름. 조용한 곳에서, 마이크 20~30cm.
    • 무성음/숨소리/자음 구간은 음정이 없어 "분석 불가"로 비워둠 — 지어내지 않음.
    • 에이전트(나)는 마이크 실측 불가 → 문법·로직만 검증. 진단이 맞는지는 오너가 직접 불러서 확인.

    🔧 어떻게 분석하나 (정직 보고)

    • 녹음: Web Audio로 원시 샘플을 직접 모음(ScriptProcessor). 압축 없는 raw PCM.
    • 분석: 녹음 전체를 25ms 간격 프레임으로 잘라 각 프레임에 YIN 피치검출(spike.html 검증 엔진 재사용) → 시간축 음정 곡선.
    • 무성음/저신뢰 프레임은 버림(구멍). 남은 유효 프레임만으로 통계.
    • 따라부르기: 목표 멜로디를 시간축에 정렬해 각 음 구간의 중앙값 cent 오차를 계산.
    • 버릇 진단: 실제 검출값의 통계(첫음 진입 오차, cent 표준편차=흔들림, 음 이동 시간, 전체 편향)로만 판단. 데이터 부족하면 "판단 불가"로 표기.
