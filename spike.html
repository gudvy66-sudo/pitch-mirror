목표 음을 골라 소리 들어보고, 같은 음을 내보세요

✅ 이 프로토타입으로 오너가 확인할 것

  • 정확한가 — 피아노/튜너앱 옆에 두고 같은 음 낼 때 음이름·Hz가 맞나
  • 반응 빠른가 — 소리 내자마자 바뀌나 (하단 지연 ms 참고)
  • 조용한 곳 vs 시끄러운 곳 — 배경소음 있을 때 숫자가 마구 튀나
  • 저음·고음 — 낮은 남성 목소리(~80~150Hz), 높은 음에서 흔들림/옥타브 오류
  • 노래할 때 — 비브라토·이어지는 음에서 안정도 미터가 얼마나 흔들리나
  • 따라부르기 — 목표음 따라 부를 때 cent 오차가 실시간으로 잘 좁혀지나

⚠️ 이 기술의 정직한 한계 (반드시 인지)

• 단선율만 됨(모노포닉). 화음/반주/여러 사람 동시엔 안 됨. 노래방 반주 깔면 망가짐 — 반주 없이 목소리만이어야 함.
• 저음이 어렵다. 남성 저음(80Hz 근처)은 주기가 길어 흔들리거나 한 옥타브 위/아래로 튈 수 있음.
• 비브라토/글리산도는 음이 계속 움직여 "정확한 한 음"으로 안 잡힘 — 안정도 미터가 이걸 보여줌.
• 마이크 품질·거리·에코에 크게 의존. 폰마다 다름. 그래서 오너가 직접 여러 기기로 테스트해야 함.
• 배경소음이 크면 오검출. 신호세기 미터가 낮은데 음이 잡히면 노이즈일 수 있음.

🔧 뭘 어떻게 만들었나 (정직 보고)

• 피치 검출: YIN 알고리즘 자체 구현(인라인). autocorrelation보다 옥타브 오류가 적어 선택. CDN 라이브러리 없음 → 오프라인·폰에서도 확실히 열림.
• 입력: Web Audio API getUserMedia + AnalyserNode(2048 샘플). 브라우저 기본 노이즈 억제/에코 제거는 끔(원음 정확도 확인 목적).
• 안정도: 최근 검출값들의 표준편차(cent)를 미터로 — 낮을수록 안정.
• 지연: 검출~화면 반영까지 걸린 프레임 시간을 ms로 표기(체감용, 정밀 측정 아님).
• 에이전트인 나는 마이크 실동작 테스트 불가 → 문법·로직만 확인함. 실제 정확도는 오너가 폰으로 확인해야 함.
